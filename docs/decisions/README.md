# Decisions & Strategic Memos

This directory contains strategic and administrative decision memos that are being reviewed, discussed, and refined for organizational and instructional direction.

## Purpose

Decision memos serve as:
- **Documentation of reasoning** – Capture the "why" behind organizational and pedagogical choices
- **Decision inputs** – Feed into the development of plans, specifications, and course materials
- **Institutional memory** – Preserve context for future decision-making
- **Discussion artifacts** – Enable collaborative refinement of strategies and approaches

## How Decisions Flow

```
Decision Memo → Discussion → Refinement → Plans & Specifications → Implementation
```

**Example flow:**
1. A decision memo proposes a new course structure or assessment approach
2. Stakeholders review and discuss implications
3. Memo is refined based on feedback
4. Decision is documented and formalized
5. Plans (course syllabi, project rubrics) are developed based on the decision
6. Specifications (assignment briefs, Excel models) are created for implementation
7. Instructors and students execute the plan

## Memo Categories

### Repository & Infrastructure Decisions
- Organizational hierarchy and directory structure
- Tools and platforms for course delivery
- GitHub workflows and integration approaches

### Pedagogical Decisions
- Course design and learning objectives
- Assessment and evaluation methods
- Integration of AI tools and emerging technologies

### Operational Decisions
- Grading policies and rubric standards
- Professional development approaches
- Documentation and record-keeping standards

## How to Use This Directory

**To propose a new decision:**
1. Create a memo using the template: `../templates/memo-template.md`
2. Name it with a date prefix: `YYYY-MM-DD-decision-title.md` for repo-wide decisions, or `YYYY-MM-DD-<course-code>-decision-title.md` for a single course/subject (e.g., `2026-05-07-bus629-stage2-restructure.md`). All decision memos live flat in this directory — no `<course-code>/` subdirectories.
3. Document the decision, rationale, and implications
4. Share for discussion and feedback
5. Refine and finalize

**To reference a decision:**
- Use relative links: `[Decision on Course Structure](./2026-02-15-repo-hierarchy.md)`
- Include decision memos in course README.md files when relevant
