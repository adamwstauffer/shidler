## Goal

Stand up (or restructure) your **public GitHub portfolio repository** before analytical work
begins. Every deliverable in this project is committed here; by the end of the term the repo is
a portfolio piece you can put on a resume. Already have a portfolio repo from BUS 313/314? You
don't start over — restructure it to the canonical skeleton and add what's missing.

## What to do

1. **GitHub account** — use your `hawaii.edu` email (unlocks GitHub Education benefits).
2. **Install GitHub Desktop** — no command line required for this course.
3. **Create the repository** — public, named professionally after *you*, not a course:
   `firstname-lastname` (or `firstname-lastname-portfolio` if taken).
4. **Build the canonical skeleton** — every folder gets a stub `README.md`:
   `README.md` (bio) · `RESUME.md` · `BIO.md` (optional) · `prompt-log.md` · `docs/`
   (`decisions/`, `specs/`, `plans/`, `templates/`) · `models/` (`templates/`, `builds/`) ·
   `data/` · `analysis/`. Do **not** copy course templates in — link to the canonical ones.
5. **Draft your bio and resume** — LLM as drafter, you as editor; log prompts in `prompt-log.md`.
6. **Commit and push** — at least **2 meaningful commits** with descriptive messages
   (e.g., `Add repo skeleton with directory READMEs`), not `update`.
