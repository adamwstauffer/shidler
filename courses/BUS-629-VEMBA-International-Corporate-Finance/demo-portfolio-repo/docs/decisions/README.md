# `docs/decisions/`

Stage 2 memos and any other decision documents. A "decision memo" captures *what* was decided, *why*, and *what alternatives were considered* — so a future reader (including yourself) can reconstruct the reasoning.

## Naming conventions

- `YYYY-MM-DD-short-title.md` — date-prefix every memo so they sort chronologically
- Use the memo template at `docs/templates/portfolio/` (or the course repo's `docs/templates/template-memo.md`)

## Structure of a decision memo

1. **Executive summary** — 100–150 words. The decision itself, up front.
2. **Context** — what problem this addresses
3. **Options considered** — at least two
4. **Recommendation** — which option, and why
5. **Risks & open questions**
