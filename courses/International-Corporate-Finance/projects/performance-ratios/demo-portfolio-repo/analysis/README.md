# `analysis/`

Self-audit and validation work. This is where you check your own models — not where the models themselves live.

## Subdirectories

| Folder | Purpose | Course stage |
|--------|---------|--------------|
| [`validation/`](validation/) | Validation reports for each model build | Stage 3 |

## What belongs here (outside `validation/`)

- Sensitivity analyses
- Scenario comparison write-ups
- Reconciliation memos when two methods produce different answers

If your audit work is short, fold it into the matching spec in `docs/specs/` instead of creating a new file here.
